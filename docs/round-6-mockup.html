Round 6 — Dynamic Input pill mockup

Two options for how multi-field input feels in M1.3 onward. Each demo shows the canvas state during an active prompt: rubber-band geometry already drawn, awaiting the user to type values OR click the cursor position. Yellow glow = focused pill (Tab cycles focus). Pills carry the values the user is editing. Hover annotations explain the elements.

How to read this: the focused pill (yellow glow) is where typed digits go. Tab cycles focus to the next pill; type → cursor caret advances; Enter commits all fields atomically.

Both options share: multi-field input, Tab cycling, per-field labels, anchored at meaningful geometric points, pills stay constant size as you zoom (chrome layer).

Full adds: witness lines (from corners to dim line), the dim line itself with arrow ticks, the angle arc + polar reference. These are painted on the canvas overlay (metric-anchored, line strokes constant px), giving the AC-style measured-dimension feel. About 30-40% more implementation work.
